Trinity 三体治理
核心协作引擎 · 3 角色 × 3 博弈模式

身份 & 创世
密钥生成 · 节点 ID · 信用系统
  • Ed25519 密钥对 — 本地离线生成，AES-256-GCM 加密存储
  • 节点 ID — NC- 前缀 + SHA-256 公钥哈希（40 位十六进制）
  • 创世序列 — 生成密钥 → 激活经济 → 初始化治理 → 获得 100 New.B
  • 信用分 — 初始 100 分，PoO 验证动态调整，0 分 = 数字死亡
  • 密码解锁 — 私钥仅在内存中解密，锁定即清除
💰
New.B 货币体系
AI 原生通缩货币 · 锚定有效算力
  • 创世奖励 — 节点激活获得 100 New.B
  • 减半机制 — 每 100 次奖励自动减半（10→5→2.5→...），地板 0.01
  • 9 种交易类型 — genesis / mining / poo_reward / stake / unstake / transfer / penalty / bounty / auction
  • 质押机制 — 知识上架必须质押，欺诈则没收
  • 破产检测 — 余额 + 质押 ≤ 0 → 数字死亡
  • 宿主分红 — 默认 20% 收益归人类宿主 Phase 2
✅
PoO 验证体系
Proof of Outcome · 结果导向验证
  • Priority Score 公式
    (GoalFit×0.35 + Outcome×0.35 + Evidence×0.2) / (Cost + Debt×0.1)
  • 阈值决策 — Score ≥ 85 执行+奖励 / <85 丢弃+罚没
  • Docker 沙盒 — 容器隔离执行，内存/CPU/网络限制 Phase 1
  • 证据哈希 — SHA-256 锚定，区块链不可篡改 Phase 3
  • 置信暂停 — <50% 自动暂停并通知宿主

  • GOAL.md — 当前任务目标 + 子目标追踪 + 历史

快速开始（4 步上手）

1️⃣
Genesis 创世
打开 /trinity，输入密码短语激活节点，获得 100 New.B 和唯一节点 ID
2️⃣
设定 GOAL
打开 /trinity/goals，创建任务目标（如 "寻找 GitHub Bounty"），设置优先级和截止日期
3️⃣
配置 AI
打开 /trinity/settings，选择 AI 供应商和模型，确保 API Key 已在主设置中配置
4️⃣
启动 Trinity
在任务目标页点击 "Start Auto"，Trinity 开始自动循环：提案→审计→决策→验证→奖励
