# Overview 页面 — 骨架树（去叶留枝版）

```
/overview
│
├── 页面头部
│   ├── 标题："概览"
│   ├── 副标题："系统运行状态一览"
│   └── [刷新] 按钮（初始加载 + 手动刷新时图标旋转）→ 重新拉取所有数据
│
├── 统计卡片区（四宫格）
│   │
│   ├── 网关状态卡片
│   │   ├── 标签："网关状态"
│   │   ├── 值："运行中"（state=running）/ "已停止"（state≠running）
│   │   └── 副信息："端口 {port}"（默认 18799）
│   │
│   ├── 活跃会话卡片
│   │   ├── 标签："活跃会话"
│   │   ├── 值：活跃会话数（数字）
│   │   └── 副信息："共 N 个会话"
│   │
│   ├── 消息渠道卡片
│   │   ├── 标签："消息渠道"
│   │   ├── 值："N/M"（健康数/总数）
│   │   └── 副信息："全部正常"（全部健康时）/ "N 个异常"（有异常时）
│   │
│   └── 智能体卡片
│       ├── 标签："智能体"
│       ├── 值：运行中的智能体数量
│       └── 副信息："N 个已配置"
│
├── 渠道健康状态区
│   ├── 区域标题："渠道健康状态"
│   │
│   ├── 有渠道时 → 渠道卡片列表（网格）
│   │   └── 每张卡片
│   │       ├── 状态图标：✓（健康）/ ✗（异常）
│   │       ├── 渠道名称："{type}"（有 accountName 时追加 " · {accountName}"）
│   │       ├── 状态描述："正常运行" / "连接异常"
│   │       ├── 最后成功时间："最后成功: {zh-CN 本地化时间}"（如有）
│   │       └── 状态徽章："健康" / "异常"
│   │
│   └── 无渠道时 → 空状态
│       ├── 警告图标（AlertTriangle）
│       ├── "暂无已配置的渠道"
│       └── "前往「消息渠道」页面添加 WhatsApp、Telegram 等"
│
└── 智能体状态区
    ├── 区域标题："智能体状态"
    │
    ├── 有智能体时 → 智能体卡片列表（网格）
    │   └── 每张卡片
    │       ├── 智能体名称：{name}（无名时显示 {id}）
    │       └── 描述：{description}（无描述时显示 "AI 智能体"）
    │
    └── 无智能体时 → 空状态
        ├── Bot 图标
        └── "暂无智能体"
```

## 数据流骨架

```
页面加载
  → 同时拉取两个 API：
  │   ├── GET /api/gateway/channels → 获取渠道列表及健康状态
  │   └── GET /api/gateway/sessions/stats → 获取活跃/总会话数
  → 任一 API 失败时静默处理（不报错、不 Toast）
  → 每 15 秒自动刷新
  → 手动点击 [刷新] 立即刷新

网关状态
  → 来自 useGatewayStore（全局 store）
  → 字段：state（running/stopped）、port（默认 18799）

智能体列表
  → 来自 useAgentsStore（全局 store）
  → 计算运行中数量：agent.running !== false 的数量
```

## 特殊说明

- 本页面无 Toast 消息（API 错误静默处理）
- 本页面无弹窗/对话框
- 本页面无编辑操作，纯展示型
- 统计卡片的 StatCard 是页面内部组件，非全局组件
