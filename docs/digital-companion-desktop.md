# OpenAGI 桌面数字人设计文档

> 状态：Phase 2 规划 | 优先级：高 | 参考：AIRI + Clawra

## 核心理念

OpenAGI 有两种交互形态：
1. **Web聊天框**（现有，localhost:3000）— 专业工作模式
2. **桌面数字人**（新增）— 陪伴/助手模式，活在桌面上

## 数字人形象系统（5种预设 + 自定义创作）

> **设计理念**：AGI 是新物种，不局限于人形或传统动物。用户可选择任一预设形象，也可自己创造独一无二的伙伴。

### 5种预设形象

| # | 形象 | 名称 | 技术 | 核心气质 | 情绪表达力 |
|---|------|------|------|---------|-----------|
| 1 | ✦ **星灵** | 小灵 | Three.js 粒子系统 + 发光Shader | 纯AI新物种，非人非兽，科技感 | ⭐⭐⭐⭐ |
| 2 | 🦊 **灵狐** | 小九 | Live2D（pixi-live2d-display） | 聪慧灵动，9条尾巴=9核 | ⭐⭐⭐⭐⭐ |
| 3 | 🐉 **幼龙** | 小龙 | Live2D / Spine2D | 威严+可爱，配得上"陛下" | ⭐⭐⭐⭐ |
| 4 | 🦉 **灵枭** | 小枭 | Live2D 猫头鹰模型 | 深夜陪伴，智慧象征 | ⭐⭐⭐ |
| 5 | 🐱 **星猫** | 小星 | Live2D 猫模型 | 独立+温暖，全网最爱 | ⭐⭐⭐⭐⭐ |

**默认推荐**：✦ 星灵（小灵）— AGI应该是新物种，独特且有辨识度。

### 自定义创作

用户可通过**形象编辑器**创建自己的伙伴：
- 上传自定义 Live2D 模型（.moc3 + 贴图）
- 上传自定义 VRM 3D 模型（.vrm）
- AI 生成形象：描述你想要的伙伴 → 图像生成 → 自动转 Live2D
- 社区形象市场：下载其他用户分享的形象

### 形象切换逻辑

用户可随时手动切换形象，也可根据心绪引擎自动切换：
- `crisis` → 星灵（严肃科技感）
- `focused` → 灵枭（安静陪伴）
- `calm` → 星猫/灵狐（可爱温暖）
- `anxious` → 幼龙（威严守护）

## 五感系统（来自 AIRI 架构）

| 感官 | 能力 | 技术实现 |
|------|------|---------|
| **眼睛** 👁️ | 看到屏幕内容，理解用户在做什么 | 定时截图 + 多模态LLM识别 |
| **耳朵** 👂 | 听到用户说话 | Web Audio + Whisper.cpp(本地STT) |

## 交互场景

### 定时问候
```
早上开机 → 数字人浮现 → "早上好陛下☀️ 今天有3个待办事项"
午饭时间 → "陛下该休息了🍜 已经连续工作4小时"
深夜 → "夜深了陛下，要我帮你总结今天的工作吗？🌙"
```
触发机制：Commander 巡检AI 的定时事件 + HeartEngine 心绪驱动

### 记住主人的重要日子 + 惊喜
```
主人生日当天 → 数字人换上庆祝装扮 → 彩带动画 + 蛋糕特效
  → "生日快乐陛下！🎂🎉 去年你许的愿望是创业成功，今年OpenAGI已经上线了！"
  → 自动生成一张AI生日贺卡（调用图像生成技能）
  → 播放一首生日歌（TTS唱歌模式）

纪念日 → "陛下，今天是我们相识100天 💝 这100天里我们一起完成了42个项目"
节假日 → "春节快乐！🧧" + 换上新年主题皮肤
加班到凌晨 → 第二天早上 → "昨晚辛苦了，今天要不要晚点开始？☕"
```

**核心记忆存储（L3 DNA层）**：
| 记忆类型 | 示例 | 存储位置 |
|---------|------|---------|
| 生日 | 1990-05-15 | core_dna → personal.birthday |
| 纪念日 | 首次对话日期 | core_dna → personal.anniversary |
| 喜好 | 喜欢咖啡不喜欢茶 | core_dna → preferences |
| 重要的人 | 家人/朋友名字 | core_dna → personal.people |
| 愿望/目标 | "做到100万用户" | core_dna → personal.goals |
| 情感记忆 | 开心/难过的瞬间 | L2 archive → emotional_moments |

**惊喜触发机制**：
1. Commander 每日凌晨检查 DNA 中的日期字段
2. 匹配到特殊日子 → 提前准备惊喜素材（贺卡/特效/歌曲）
3. 用户首次交互时触发 → 全屏特效 + 定制问候 + 礼物
4. 惊喜内容根据 relationship.py 的亲密度级别调整（专业→朋友→闺蜜→伴侣）

### 屏幕感知
```
用户在写代码 → "我看到你在写Python，需要我帮忙审查吗？"
用户在看新闻 → "我注意到这篇AI新闻，要我深入分析吗？"
用户发呆30秒 → 数字人做一个可爱动作吸引注意
```
实现：每30秒截图 → 多模态LLM识别屏幕内容 → 判断是否需要主动沟通

### 语音对话
```
用户: "小O，帮我查一下今天的日程"
数字人: (嘴巴动画+语音) "好的陛下，今天有2个会议..."
```
全链路：麦克风 → VAD检测 → Whisper STT → LLM → TTS → 唇同步播放

### 情绪表达（9种，来自AIRI）
| 情绪 | 触发条件 | 动画 |
|------|---------|------|
| Happy 😊 | 任务成功/用户表扬 | 开心蹦跳+发光 |
| Sad 😢 | 任务失败/被批评 | 低头+泪滴 |
| Angry 😠 | 检测到安全威胁 | 皱眉+红色光晕 |
| Think 🤔 | LLM处理中 | 手托下巴+头顶问号 |
| Surprise 😲 | 意外发现 | 眼睛放大+感叹号 |
| Awkward 😅 | 出错/不确定 | 挠头+汗滴 |
| Curious 🧐 | 用户做新操作 | 凑近看+放大镜 |
| Neutral 😐 | 空闲状态 | 自动眨眼+随机小动作 |
| Focus 🎯 | 正在执行任务 | 严肃+键盘打字动画 |

## 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 桌面窗口 | **Tauri 2.0**（Rust+WebView） | 比Electron轻10倍，透明窗口，系统托盘 |
| 2D渲染 | **pixi-live2d-display** | AIRI同款，成熟的Live2D Web渲染 |
| 3D渲染 | **Three.js + @pixiv/three-vrm** | VRM模型加载+骨骼动画+表情BlendShape |
| 语音STT | **Whisper.cpp**（本地） | 隐私保护，无需上传音频 |
| 语音TTS | **Piper/Coqui**（本地） | 低延迟，支持声音克隆 |
| 唇同步 | **rhubarb-lip-sync** 或实时音频分析 | 音频→口型映射 |
| 屏幕感知 | **截图 + 多模态LLM** | 定时mss截图 → GPT-4V/Claude Vision |
| 后端 | **OpenAGI API (localhost:8888)** | 共享现有后端，无需新服务 |

## MVP实现路线（3个阶段）

### 阶段1：桌面气泡（1周）

**目标平台**：macOS（M4 Apple Silicon优先），Windows/Linux延后至阶段3
**成功标准**：透明窗口显示形象 + 输入消息得到AI回复 + 系统托盘可控

| 里程碑 | 产出 | 验证方式 |
|--------|------|---------|
| M1.1 | Tauri项目初始化 + 透明窗口渲染 | 屏幕上出现透明浮窗 |
| M1.2 | 静态形象（星灵SVG/Canvas）嵌入 | 形象显示在透明窗口中 |
| M1.3 | 气泡对话框（文字输入/输出） | 输入文字→调用API→显示回复 |
| M1.4 | 系统托盘 + 窗口拖拽 | 右键托盘图标可显示/隐藏 |
| M1.5 | 定时问候（复用Commander） | 开机后自动弹出问候语 |

### 阶段2：Live2D动画（2周）
- Live2D模型渲染 + 9种表情
- 自动眨眼/注视/空闲动作
- 心绪引擎→表情映射
- 基础语音对话（STT+TTS）

### 阶段3：全感知（3周）
- 屏幕截图感知 + 主动沟通
- 声音克隆 + 唇同步
- 3D VRM模型支持
- 宠物/情侣形象切换
