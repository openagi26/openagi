> 里程碑：M1-B1  
> 日期：2026-04-10  
> 设计师：用户体验和界面设计师  
> 原型文件：`docs/prototypes/trinity-management-wireframe.html`

---

## 一、设计目标

为 Trinity 三体治理系统设计统一的管理页面，整合三大核心模块：
1. **身份信息展示** — 节点身份、信用、安全状态
2. **三AI协作状态** — 扩张者/风控员/财务官的实时协作流程
3. **治理账本概览** — 证据链、战略债务、PoO验证、操作时间线

---

## 二、页面布局

### 三栏式布局（1440px 最大宽度）

```
┌─────────────────────────────────────────────────────┐
│  页面头部：标题 + 节点ID + 状态徽章 + 模式标识      │
├─────────────────────────────────────────────────────┤
│  经济概览条：余额 | 总收入 | 总支出 | 已质押         │
├──────────┬────────────────────┬─────────────────────┤
│ 左栏     │ 中栏               │ 右栏                │
│ 280px    │ 弹性               │ 320px               │
│          │                    │                     │
│ 身份头像 │ 三AI角色卡（3列）   │ 证据链摘要          │
│ 身份详情 │ 协作流程管道        │ 战略债务列表        │
│ 信用分数 │ 游戏模式选择        │ PoO 验证统计        │
│ 置信度条 │ 最近循环结果        │ 操作时间线          │
│ 操作按钮 │                    │ 成功经验库          │
│          │                    │                     │
└──────────┴────────────────────┴─────────────────────┘
```

### 响应式断点
- `> 1200px`：三栏布局
- `900-1200px`：左栏 + 中栏，右栏下方双列展示
- `< 900px`：单列堆叠

## 三、核心模块设计

### 3.1 身份信息面板（左栏）

| 组件 | 说明 |
|------|------|
| 身份头像 | 渐变紫色圆形头像，右下角绿色状态点 |
| 节点ID | 等宽字体，截断显示 |
| 信息行 | 状态/创建时间/总轮次/待处理债务，键值对排列 |
| 信用分数 | SVG 圆环进度，数值居中，视觉突出 |
| 置信度条 | 水平进度条，颜色随值变化（绿/黄/红） |
| 操作按钮 | 解锁身份（主要）/ AI人格设置（轮廓）/ 锁定身份（危险） |

**数据源 API**：
- `GET /api/trinity/identity` — 身份信息
- `POST /api/trinity/identity/unlock` — 解锁
- `POST /api/trinity/identity/lock` — 锁定
- `GET /api/trinity/dashboard` — 信用分、置信度

### 3.2 三AI协作面板（中栏）

| 组件 | 说明 |
|------|------|
| AI角色卡（×3） | 顶部色条区分角色，图标+名称+状态+实时输出 |
| 协作流程管道 | 5步流程：目标解析→扩张提案→风控审计→财务裁决→PoO验证 |
| 游戏模式选择 | 三选一卡片：辩论/竞争/精炼 |
| 最近循环结果 | 最近5轮得分，颜色编码（绿/黄/进行中） |

**AI角色色彩系统**：
| 角色 | 主色 | 色值 | 语义 |
|------|------|------|------|
| AI-1 扩张者 | 红-橙 | `#f87171` | 激进、行动 |
| AI-2 风控员 | 青-蓝 | `#22d3ee` | 冷静、审视 |
| AI-3 财务官 | 琥珀-黄 | `#fbbf24` | 理性、价值 |

**状态指示**：
- `active` — 绿色圆点 + 文字
- `thinking` — 紫色脉冲圆点 + 动画
- `waiting` — 灰色静态圆点

**数据源 API**：
- `GET /api/trinity/state` — 协作状态、当前阶段
- `GET /api/trinity/history` — 循环历史
- `PUT /api/trinity/game-mode` — 模式切换
- `GET /api/trinity/roles/config` — AI人格配置

### 3.3 治理账本面板（右栏）

| 组件 | 说明 |
|------|------|
| 证据链摘要 | 证据等级徽章（H1-H4）+ 描述 + 时间 + 来源 |
| 战略债务 | 严重等级 + 描述 + 成本 + 解决按钮 |
| PoO统计 | 2×2 网格：总任务/已验证/失败/平均分 |
| 操作时间线 | 竖向时间线，圆点颜色区分类型 |
| 成功经验库 | 经验标题 + 成功率，紧凑列表 |

**证据等级色彩**：
| 等级 | 颜色 | 语义 |
|------|------|------|
| H1 | 红色 | 风险警告/未验证 |
| H2 | 黄色 | 间接证据 |
| H3 | 蓝色 | 实证数据 |
| H4 | 绿色 | 高度可信 |

**数据源 API**：
- `GET /api/trinity/governance/evidence` — 证据链
- `GET /api/trinity/governance/debts` — 战略债务
- `POST /api/trinity/governance/debts/:id/resolve` — 解决债务
- `GET /api/trinity/poo/stats` — PoO统计
- `GET /api/trinity/governance/playbooks` — 成功经验

## 四、设计规范

### 颜色系统
沿用 OpenAGI "星空紫" 主题：
- 主色：`#a855f7` (purple-500)
- 卡片背景：`rgba(30, 24, 56, 0.6)` + `backdrop-filter: blur(20px)`
- 边框：`rgba(139, 92, 246, 0.15)`
- 文字层次：primary `#f0ecf9` → secondary `60%` → muted `35%`

### 组件状态
- `glass-card`：基础卡片，hover 时边框亮起
- `badge`：圆角药丸状，颜色语义化
- `button`：primary（渐变紫）/ outline（边框）/ danger（红色）

### 圆角
- 大卡片：`12px`
- 中等元素：`8px`
- 小元素/徽章：`6px` / `20px`

### 间距节奏
- 卡片间距：`16-20px`
- 内边距：`16-20px`
- 信息行间距：`10px`

## 五、交互说明

1. **游戏模式切换**：点击模式卡片切换，选中态高亮
2. **债务解决**：点击"解决"按钮，卡片淡出+右移动画
3. **身份解锁/锁定**：按钮触发密码输入弹窗
4. **AI 实时状态**：thinking 状态有脉冲动画
5. **协作流程**：当前步骤高亮，已完成步骤绿色，待执行灰色

## 六、与现有页面的关系

本原型是对现有 Trinity 页面的**管理视图增强**：

| 现有页面 | 本原型定位 |
|----------|-----------|
| `Trinity/index.tsx` | 被本原型替代（Dashboard首页） |
| `Trinity/GovernancePage.tsx` | 治理详情页（从本原型跳转） |
| `Trinity/GoalPage.tsx` | 目标管理页（从本原型跳转） |
| `Trinity/SettingsPage.tsx` | 设置页（不变） |

### 建议路由
- `/trinity` → 本原型（管理首页/Dashboard）
- `/trinity/governance` → 治理详情（保留现有）
- `/trinity/goals` → 目标管理（保留现有）

## 七、前端实现建议

1. **组件拆分**：将三栏拆分为 `<IdentityPanel />` / `<TrinityCollaboration />` / `<GovernanceOverview />` 三个组件
2. **实时更新**：协作流程和AI状态建议 WebSocket 或 3s 轮询
3. **动画**：使用 Tailwind 的 `transition-all` + CSS keyframes
4. **数据获取**：复用 `useTrinityStore` + `hostApiFetch`
