# RobotDuck借鉴研究报告 — 三方讨论汇总

> 2026-04-16 | 3个独立讨论代理(sonnet+haiku+opus)

## 一、产品视角（Sonnet）

### 功能迁移表

| 功能 | 迁移性 | 方案 |
|------|--------|------|
| 实时语音对话（FunASR+通义千问） | **100%可迁移** | 已有Whisper/Vosk/FunASR |
| 语音合成（CosyVoice多音色） | **100%可迁移** | 接入CosyVoice API或edge-tts |
| 视觉问答（通义千问VL流式） | **100%可迁移** | 屏幕截图+摄像头→VL模型分析 |
| 音色克隆（录几秒即可） | **90%可迁移** | CosyVoice克隆接口 |
| 手势检测（MediaPipe） | **需适配** | 改为手势触发：挥手唤醒/捂嘴静音/竖拇指赞同 |
| 表情动画（16通道舵机） | **概念迁移** | 物理舵机→粒子情绪物理动画 |
| 机械臂控制 | **暂不适用** | 保留接口给未来机器人版 |

### 创新建议TOP3

1. **屏幕感知+实时注释** — 小星不只"被问"，而是持续观察屏幕主动说"那行代码有拼写错误"
2. **声音克隆×情境切换** — 用家人声音做温暖提醒，比机器音更私人化
3. **专注模式+生理节律感知** — 结合摄像头判断用户是否发呆，动态调整打扰频率

### 物理机器人架构

```
小星桌面核（大脑）↔ WebSocket ↔ 机器人边缘节点（身体）
```
桌面小星做"云端大脑"，机器人做"轻量执行体"，断网时降级为本地反射模式。

## 二、技术视角（Haiku）

### 技术映射表

| RobotDuck | 小星 | 说明 |
|-----------|------|------|
| 舵机动画 | 粒子变形+运动轨迹 | Trinity驱动粒子形变 |
| 摄像头→MediaPipe | 屏幕截图+摄像头→分析 | 前端Tauri截屏/拍照 |
| 阿里云FunASR | Vosk/FunASR/Whisper | 已有多引擎STT |
| CosyVoice音色克隆 | edge-tts+情绪引擎 | 多引擎TTS |
| 通义千问VL | 多模态LLM Router | 视觉问答通过LLM路由 |

### 核心架构：ActuatorInterface抽象层

```
统一指令格式：{emotion, intensity, duration, target_shape}
├─ Desktop: ParticleActuator（粒子渲染）
└─ Robot: ServoActuator（舵机控制）
```

RobotDuck和小星共享指令生成逻辑，仅驱动层差异。

### 新增API端点

- `POST /api/animate/emotion` — 情绪→粒子动画
- `POST /api/vision/ask` — 截图/拍照+问题→视觉回答
- `POST /api/robot/servo-sync` — ESP32伺服同步（预留）

### 技术风险TOP3

1. ESP32↔Tauri通信延迟 — 舵机需<200ms → 本地缓冲+降级
2. ChromaDB+粒子渲染内存争用 — 分离进程
3. 实时手势检测性能 — 降低检测频率

## 三、战略视角（Opus）

### 商业路径

**先桌面后硬件**，桌面端零硬件成本、快速迭代。硬件在10万用户后启动，用桌面用户做种子众筹。

### 云策略

**多云策略，不绑阿里**。国内用阿里（合规+成本），海外用OpenAI/Anthropic。核心是抽象统一接口层。

### 音色克隆获客价值

中等偏高。方言模式在下沉市场有传播力，作为"惊喜功能"而非主打卖点。

### 创新功能TOP3

1. **跨设备记忆连续体** — 桌面聊的内容，机器人无缝接续
2. **主动感知+预判行动** — 不等指令，根据屏幕/日程/情绪主动服务
3. **用户人格镜像** — 学习用户说话风格，成为"另一个我"

### "一魂多体"商业模型

小星作为云端人格核心，桌面/机器人/机器狗共享同一记忆和性格。按载体收费：桌面免费引流，硬件订阅制。这是100万到1000万的杠杆点。
