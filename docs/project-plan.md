## 三、推进计划（4个阶段）

### 阶段 A：质量基线验证（第1轮，优先级最高）

**目标**：确认现有代码可编译、可运行、质量达标。

| 编号 | 任务 | 负责人 | 交付物 |
|------|------|--------|--------|
| A1 | 运行 `pnpm run lint` 检查代码规范 | 质检 | lint 报告 |
| A2 | 运行 `pnpm run typecheck` 类型检查 | 质检 | 类型错误清单 |
| A3 | 运行 `pnpm test` 单元测试 | 质检 | 测试结果报告 |
| A4 | 人工审查 v6.0 新增模块代码质量 | 代码审查 | 代码审查报告 |
| A5 | 评估 trinity/governance/identity/newb/poo 模块成熟度 | 后端 | 模块成熟度评估 |

**质量门控**：lint 零错误、typecheck 通过、测试通过率 > 95%。

### 阶段 B：问题修复与稳定（第2轮）

**目标**：修复阶段A发现的所有问题，确保基线稳定。

| B1 | 修复 lint 错误 | 后端 | A1 |

| B3 | 修复失败测试 | 后端 | A3 |
| B4 | 根据审查报告优化代码 | 后端/前端 | A4 |

| B6 | 确认 `pnpm dev` 可正常启动 | 质检 | B5 |

## 五、协作机制

1. **每轮开始**：项目统筹发布本轮任务清单，明确优先级和依赖关系
2. **任务领取**：各角色按分工执行，遇阻塞立即上报
3. **每轮结束**：质检验证 → 代码审查确认 → 项目统筹汇报给您
4. **决策上报**：范围变更、技术选型、优先级调整需上报高级项目经理
5. **流程追踪**：工作流程管家在任务看板中维护所有任务状态
